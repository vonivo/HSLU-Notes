Acceptance test-driven development ist eine ertweiterung von TDD.

1. Before Coding -> Defines Acceptance criteria as executable tests.
2. Mache gleich weiter wie mit TDD-Entwicklung

TDD-Reglen:
- Write only so much Code until a unit test fails.
- Write only so much Code to make the failing test pass.
- Refactor only when all tests pass.
