Wenn ein Code-Review durchgeführt wird, sollte man immer freundlich und konstruktiv sein.

- Code Lesbarkeit
	- Ist der Code selbsterklärend
	- Ist der Code verständlich
	- Sind die Namen aussagekräftig
	- Ist die Dokumentation ausreichend
	- Sind die Clean Code Developer Prinzipen beachtet
- Test
	- Sind die Tests verständlich
	- Gibt es Lücken in der Testabdeckung
- Was müsste geändert werden damit die Verantwortung für den Code übernommen werden kann.
