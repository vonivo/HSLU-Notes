Es gibt zwei Arten von Compiler:
- **One Tier**: Von Programm-Code direkt zu Assembly/Binary.
- **Tow-Tier**: Von Programm-Code -> Vm-Code ->Assembly/Binary

Der Jack-Compiler sieht wie folgt aus:
![[Pasted image 20260619150511.png]]

- Der Jack-Compiler kompiliert jedes `.jack`-File in ein einzelnes `.vm` File.
- Jedes Unterprogramm `subName` in einer Datei `ClassName.jack` wird in eine [[Funktionsbefehle|VM-Funktion]] names `ClassName.subName` kompiliert.
- Konstanten:
	- `true` wird in VM-Code als `constant 1` dargestellt, gefolgt vom `neg`
	- `false` wird im VM-Code als `constant 0` dargestellt.
	- `null` wird im VM-Code als `constant 0` dargestellt.
	- `this` wird im VM-Code als `pointer 0` dargestellt.
- Lokale Variablen werden auf `local 0, local 1, ...` abgebildet.
- Argument Variablen werden auf `argument 0, argument 1, ...` abgebildet.
- Statische Variablen werden auf `static 0, static 1, ...` abgebildet.
- Feld Variabeln werden auf `this 0, this 1, ...` abgebildet.
- Der Compiler behandelt:
	- Multiplikation `*` als `Math.multiply()`
	- Division `/` als `Math.divide()`
	- String-Konstanten durch `String.new(length)`
	- String-Zuweisung wie `x="cc"` durch eine Folge von Aufrufen von `String.appendChar(c)`

### [[Tokenizing|Tokenizer]]
Der Tokenizer bietet Dienste für:
- Überspringen von Leerzeichen
- Ermittel des aktuellen Token und Weiterschalten zur Eingabe danach
- Ermitteln des Typs des aktuellen Tokens
