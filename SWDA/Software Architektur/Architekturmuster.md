>[!Definition]
>Architekturmuster beschreiben als Konzpet den Grundaufbau eines ganzen System. (Vergleichbar mit [[Entwurfsmuster]])

Architekturmuster sind **nicht so stark vereinheitlicht** und 'standardisiert' wie beispielsweise die Entwurfsmuster (GoF-Patterns)

## Beispiele:
- **Domain Logic Pattern:** Domain Model, Service Layer
- **Data Source Architectural Pattern**: Data Mapper, etc.
- **Object-Relational Behavioral Pattern:** Unit of Work, etc.
- **Object-Relational Metadata Pattern:** Repository, etc.
- **Web Presentation Pattern:** Model View Controller, Page Controller, Front Controller, etc.
- **Distribution Patterns**: Remote Facade, Data Transfer Objects, etc.
- **Base Patterns**: Gateway, Mapper, Registry, Value Objects, Special Case, Plugin, Service-Stub
