API-Patterns werden verwendet, um in [[Application Programming Interface|API]]s die Namensgebung und das Verhalten möglichst vorhersehbar und durchgängig zu gestalten.

- **Repetition**: Verwende immer gleiche Namen für gleiche Dinge.
- **Periodisch**: Verwende immer gleiche Bezeichnung für gleiches Verhalten.
- **Symmetrie**: Biete, wenn möglich symmetrische Methoden an, z.B. für das Öffnen und Schliessen, oder das Belegen und Freigeben.
