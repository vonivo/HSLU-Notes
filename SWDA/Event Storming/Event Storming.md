Im Event Storming werden zuerst alle möglichen [[Domain Event|Domain-Events]] gestormt.

Wenn man diese hat, werden dazu 

1. Commands erfasst.
2. Daten zu den Events und Commands erfasst.
3. Definieren von Views / APIs über die Akteure Befehle ausführen.
4. Definieren der [[Bounden Context|Bounded Contexts]]
5. Dokumentation der Businessprozesse. Z.B. mit BPMN
6. Erstellen und Priorisieren von [[Epic und Feature|Epics]]
7. Erstellen von [[User Story]]
